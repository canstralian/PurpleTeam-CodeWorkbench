# Security Policy

## Supported Versions

Only the latest version of Purple Team Code Workbench is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please report it responsibly.

Do **NOT** open a public issue for security vulnerabilities. Instead, please follow these steps:

1.  Send an email to [your-security-email@example.com] (replace with actual email if available).
2.  Provide a detailed description of the vulnerability.
3.  Include steps to reproduce the issue.
4.  Provide any potential mitigation strategies.

We will acknowledge your report and work to address the issue as soon as possible.

## Intended Use & Safety

Purple Team Code Workbench is a tool for **authorized security testing and research**. 

Users are responsible for:
- Obtaining proper authorization before using this tool on any system.
- Complying with all applicable laws and regulations.
- Validating all AI-generated outputs before operational use.

The project maintainers are not responsible for any misuse or damage caused by this tool.
